## 목적

캐릭터/파티 시스템은 Project Aether의 핵심 조작 경험입니다. 플레이어가 여러 캐릭터를 전환하면서도 전투 상태가 자연스럽게 이어지는 구조를 보여주는 것이 목표입니다.

---

## 보여주려는 역량

| 항목 | 설명 |
|------|------|
| 캐릭터 상태 관리 | 캐릭터별 쿨다운, 에너지, 상태 태그를 분리해 관리합니다. |
| 전환 생명주기 | 입력 정리, 위치/속도 전달, 카메라 흐름, 활성 캐릭터 갱신을 하나의 과정으로 다룹니다. |
| 전환 정책 | 전환 가능한 Ability와 전환을 막아야 하는 Ability를 구분합니다. |
| 비용 제어 | 활성 캐릭터와 비활성 캐릭터의 처리 비용을 다르게 관리합니다. |

---

## 설계 질문

| 질문 | 포트폴리오에서 설명할 내용 |
|------|--------------------------|
| 왜 캐릭터별 상태를 유지하는가? | 파티 전환 후에도 쿨다운, 에너지, 상태가 보존되어야 하기 때문입니다. |
| 전환 중 입력은 어떻게 처리하는가? | 이전 캐릭터의 입력이 새 캐릭터로 새지 않도록 전환 시점에 정리합니다. |
| 어떤 상황에서 전환을 막는가? | 중요한 공격 연출, 피니시 모션, 취소 불가 상태에서는 전환을 제한합니다. |

---

## 완료 기준

- 4인 파티 슬롯을 전환할 수 있습니다.
- 전환 후에도 캐릭터별 상태가 유지됩니다.
- 전환 가능/불가능 상태가 명확히 구분됩니다.
- 전환 과정이 HUD와 플레이 흐름에 자연스럽게 반영됩니다.
