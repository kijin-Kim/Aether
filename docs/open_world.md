## World Partition & Navigation

### 구현 규모 및 시나리오

- **맵 규모**: 총 4km² (2km × 2km). 초원·해안가·고대 유적 3가지 지상 바이옴 + Z축 방향으로 복층형 지하 던전 1구역
- **에셋 배치**: World Partition Data Layer로 지상/지하/이벤트 분리 구성
- **스트리밍 사양**: 256×256m 단위 그리드 셀 로드/언로드

### 핵심 기능 및 기술 아키텍처

| 기술 요소 | 설계 내용 | 구현 방식 |
|-----------|-----------|----------|
| **World Partition** | 맵 전체를 256×256m 그리드 셀로 분할. 카메라 기준 로딩 반경 내 셀만 런타임 로드 | PlayerController를 스트리밍 소스로 사용. 패스트트래블 시에는 동기 로드(로딩 화면) |
| **Data Layer** |  플레이어 고도에 따라 특정 Layer만 선택적 로드/언로드 |  |
| **HLOD 레이어** | 언로드된 셀 콘텐츠를 저해상도 메시로 원거리 렌더링 |  |
| **스플라인 에디터 툴** | 기획 반복 작업 단축용 스플라인 기반 자동 오브젝트 배치 툴 | 스플라인 경로를 따라 윈드 필드와 식생을 HISM으로 배치 |
